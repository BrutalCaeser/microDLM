🎲 Token Masking Process

How tokens are randomly masked during training with the cosine schedule

How It Works

During training, tokens are randomly masked with a probability determined by the cosine schedule α(t) = cos(πt/2)². At t=0, no tokens are masked. At t=1, all tokens are masked. This creates a curriculum from easy to hard denoising tasks.

Key Difference

Diffusion models use bidirectional attention - each token can attend to all other tokens in the sequence. GPT uses causal attention - each token can only attend to previous tokens. This fundamental difference enables parallel generation in diffusion vs sequential generation in GPT.

Key Insight

Diffusion computes loss only on masked positions (typically ~50% of tokens per batch). This makes the training signal "harder" but acts as a natural regularizer.

GPT computes loss on all positions, giving it a "richer" training signal but making it prone to overfitting without regularization.

Performance Comparison

Diffusion generates ~6× faster by revealing multiple tokens in parallel based on confidence. It takes only 5 steps to unmask 5 tokens.

GPT generates sequentially, one token at a time. It takes 5 steps to generate 5 tokens, but each step is faster than diffusion steps.

The key advantage of diffusion is parallelism - it can process multiple positions simultaneously.

How It Works

The cosine schedule α(t) = cos(πt/2)² determines how many tokens to unmask at each step:

  • At t=0: α=1, no tokens masked (clean text)
  • At t=1: α=0, all tokens masked (pure noise)
  • During generation: Tokens are unmasked according to the schedule's derivative

This creates a smooth progression from noise to signal, with more tokens unmasked in the middle steps where the model is most confident.

🎯 Confidence-Based Unmasking

How diffusion models reveal tokens based on prediction confidence

50%

Key Concept

Diffusion models compute confidence scores for each masked token prediction. Tokens with confidence above a threshold are revealed simultaneously, enabling parallel generation. High-confidence predictions (green) are revealed first, followed by medium (yellow) and low (red) confidence predictions.

This approach allows diffusion models to generate multiple tokens in parallel while maintaining quality, unlike GPT's sequential left-to-right generation.

Key Difference

⚖️ Overfitting vs Generalization

Why diffusion models generalize better than autoregressive models

Key Insight

Diffusion Models have built-in regularization through random masking, which forces the model to learn robust representations that work across different corruption levels. The position-specific loss computation prevents the model from memorizing sequences.

GPT Models are prone to overfitting because they process sequences autoregressively, leading to target leakage where later tokens can indirectly "see" earlier targets through the model's internal representations. Without explicit regularization, they tend to memorize training data.

This fundamental difference explains why diffusion models can achieve better generalization with the same architecture and training data.

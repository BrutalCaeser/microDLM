The Same Bones

Both models are 6-layer bidirectional transformers with identical building blocks: token embeddings, RoPE positional encoding, RMSNorm, multi-head attention with QK-norm, and ReluSquared feed-forward networks.

The architecture is the same. The difference is in how they see and what they predict.

diff #1 · vocabulary

The Mask Token

Diffusion adds a special [MASK] token (id 0) to the vocabulary. This is how the model represents "I don't know yet" — a position that needs to be filled in.

GPT has the same vocab size for a fair parameter comparison, but never actually uses the mask token. It doesn't need one — it generates left-to-right and never sees "unknown" positions.

# diffusion.py
MASK_CHAR = "_"
chars = [MASK_CHAR] + chars  # index 0 = MASK
vocab_size = 66  # 65 chars + 1 MASK

# gpt.py
chars = [MASK_CHAR] + chars  # same size,
# but MASK is never used during training or generation

diff #2 · the key difference

Causal vs Bidirectional

One line of code. Everything changes.

GPT — is_causal=True

Each position can only attend to tokens before it. The model never sees the future. This is why it must generate one token at a time, left to right.

F.scaled_dot_product_attention(q, k, v,
  is_causal=True)  # ← causal mask

Diffusion — is_causal=False

Each position attends to all positions — past, present, and future. The model sees the full context, including masked tokens, to predict what goes where.

F.scaled_dot_product_attention(q, k, v,
  is_causal=False)  # ← bidirectional
diff #3 · training data

Next-Token vs Random Masking

GPT sees x[i] and predicts x[i+1]. Simple shift — input and target are offset by one position.

Diffusion takes a clean sequence, randomly masks a fraction of tokens (uniform rate from 0% to 100%), and asks the model to predict the originals at masked positions.

GPT: next-token shift

input x[i] → target x[i+1]
↓ predict next

Diffusion: random masking

mask ~50% → predict originals
↓ predict masked
diff #4 · loss function

Where to Compute Loss

Both use cross-entropy. The difference is which positions count.

GPT — all positions

Every token contributes to the loss. The model learns from predicting every next token in the sequence. More signal per batch.

# Loss over ALL positions
loss = F.cross_entropy(
  logits.view(-1, V),
  targets.view(-1)
)

Diffusion — masked positions only

Only masked tokens contribute. Unmasked tokens are already known — predicting them would be trivially easy and uninformative.

# Loss only on masked positions
loss = F.cross_entropy(..., reduction="none")
loss = (loss * mask).sum() / mask.sum()
diff #5 · generation

Sequential vs Parallel

This is where the architectural differences become visible.

GPT — left to right, one at a time

Generate token 1, append, generate token 2, append... Each step produces exactly one new token. T tokens = T forward passes.

Diffusion — parallel iterative unmasking

Start from all [MASK]. Each step predicts all positions, unmasks the most confident batch. T tokens in N steps (N ≪ T).
